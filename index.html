Description

This project computes mathematical operations without using Scratch's built-in Operators category. The engine reconstructs arithmetic, comparisons, and mathematical functions entirely using custom logic.

It is designed for understanding computation internals, experimenting with dependency-based algorithms, and exploring how complex math emerges from simple rules.

Educational system Structural simulation Custom computation layers No native operators
Compact overview: the project behaves like a hand-built arithmetic model with explicit dependencies, rollback protection, and layered comparison logic.

Safety and Stability Improvements

Buffer / rollback system

Before modifying any shared variable: previous value is stored in a backup variable After execution: state is restored

What it prevents

accidental overwrites cross-function contamination recursive state corruption overwrite conflicts in chained calls
Don't forget to look at profile !

Remaining Risks

Even with the buffer system, recursion in division remains sensitive, deep dependency chains can become hard to debug, floating approximation errors may accumulate, and large chained calls may increase execution time.

Deep recursion sensitivity Approximation drift Slower large chains Not parallel-safe Debug complexity
